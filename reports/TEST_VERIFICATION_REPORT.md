## 发现的问题

### 1. UI 组件测试配置问题 (高优先级)
**影响范围**: 5 个 UI 组件测试文件  
**问题描述**: vitest 配置中 vue-i18n 插件未正确安装，导致使用 `useI18n()` 的组件测试失败  
**影响测试**: 
- cron-editor.test.ts (14 个用例)
- theme-switcher.test.ts (8 个用例)
- dashboard-card.test.ts (6 个用例)
- smart-search.test.ts (9 个用例)

**建议修复**:
```javascript
// vitest.config.ts
import { createI18n } from 'vue-i18n'

// 在 setup 文件中添加
beforeEach(() => {
  const i18n = createI18n({ legacy: false })
  app.use(i18n)
})
```

### 2. Pinia Store Ref 访问问题 (中优先级)
**影响范围**: cron-store.test.ts (11 个用例失败)  
**问题描述**: 测试中直接对 store 的 Ref 属性赋值，但该 Ref 可能为 null  
**建议修复**: 使用 store 暴露的方法而非直接操作 `.value`

### 3. 代码覆盖率偏低 (低优先级)
**当前覆盖率**:
- 语句覆盖：2.34%
- 分支覆盖：1.36%
- 函数覆盖：5.90%

**原因**: 测试仅覆盖了 3 个 store (auth, notification, rbac)，大部分 store 和 server 端代码无测试

**建议补充测试的模块**:
- `session.ts` (会话管理)
- `cron.ts` (定时任务)
- `config.ts` (配置管理)
- `agent.ts` (Agent 管理)
- `backup.ts` (备份功能)
